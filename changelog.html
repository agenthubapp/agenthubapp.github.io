Changelog

What's new in AgentHub

A running log of releases, features and fixes. Newest first.

v0.5.1 — Electron PiP edge dock Latest Jun 25, 2026

Pop out → dock ngay.

  • Pop out → dock ngay — mở thẳng tab mép màn hình (không flash panel đầy)
  • Kéo tab / panel — trượt dọc mép trái/phải hoặc ngang mép trên/dưới; thả tay tự snap về mép gần nhất (giống PiP macOS)
  • Smart peek — tự bung panel khi có action (plan ready / needs input), 5s không tương tác thì thu lại
  • <b>Tap chevron</b> để expand; header drag khi expanded
  • pack:install — build + copy thẳng vào /Applications
v0.5.0 — Cursor planning & agent clarity Jun 25, 2026

Cursor plans reliably; you always know who executes.

  • Cursor headless planning — strict micro-plan prompt, auto-retry when the CLI returns scout prose instead of a ## Scope plan (up to 4 attempts).
  • Execute visibility — banner + cockpit show the active run's agent and model; Assign-agent dropdown syncs with the open run.
  • Switch agent — defaults to current executor; Apply model on awaiting_plan without forced Codex swap.
  • Mobile ticket UX — compact header, sticky tabs, more scroll room on Run/Changes/Context.
v0.4.2 — Ask workspace & mobile UX Jun 25, 2026

Smarter Ask, cleaner mobile, board that remembers your view.

  • Ask workspace — unified search, task orchestration, screenshot annotate, send reply to active run, @file mentions, and conversational memory across chats.
  • Fast Ask without Ollama — instant ticket summaries instead of slow CLI spawn; no more leaked system prompts in replies.
  • Chat polish — typewriter replies, thinking dots, scrollable history, board view preserved when backing out of a ticket.
  • Mobile nav — 5 daily-driver tabs + More sheet (Autopilot, Terminal, Metrics, Wiki, Settings); search on header.
  • Jira sync fix — stale tickets pruned correctly after reassignment when the board is full.
v0.4.1 — UI hotfix Jun 22, 2026

Layout fixes for Mission Control and the board.

  • Inspector sidebar — no longer overlays ticket detail; sits beside content on wide screens.
  • Board card labels — triage and priority chips wrap instead of stacking on top of each other.
  • Ticket detail scroll — flex height chain + mobile bottom inset so you can scroll to the bottom reliably.
v0.4.0 — Mission Control & agent intelligence Jun 22, 2026

A control room built for how you actually ship.

  • Mission Control workspace — board or list, split detail + live preview + inspector, run cockpit, and per-ticket base branch.
  • macOS-native shell — vibrancy, SF Pro, Liquid Glass sidebar, grouped nav, ⌘K command palette, and shortcut cheatsheet.
  • Today briefing — a daily feed of what needs you, wins, and an LLM-written digest from your project activity.
  • Agent Activity cockpit — live routing, activity feed, sub-agent tree, and phase timeline so runs are observable, not opaque.
  • Cursor CLI as a third executor alongside Codex and Devin, with smart fast/deep routing per phase.
  • Ask upgrades — cross-session memory, review code → file Jira tickets (AgentHub POSTs issues directly), visual answers with screenshot + Figma.
  • Conflict resolution — PR mergeable state on GitHub, push/force-push, manual 3-way hunk picker, and agent-assisted resolve with marker guards.
  • Spec Steward — decision book, spec contract, and auto-resolve for open plan questions so agents stay on-spec across tickets.
  • Learn-from-edits loop — when you edit a merged PR, AgentHub distills corrections into wiki learnings for future runs.
  • Maestro regression sentinel, PiP pop-out panel, Android stack support, and ~25–50× faster warm RAG retrieval via vector cache.
v0.3.1 Jun 2026

Packaging polish.

  • Terminal in the .app — ships node-pty so the worktree terminal works in the packaged build.
  • Mobile terminal proxy — /terminal-ws through the web port so Tailscale HTTPS can reach the worker PTY.
v0.3.0 — Ask, terminal & notifications Jun 2026

Faster answers, deeper mobile access.

  • Ask fast path — simple questions answer in under a second; deep questions route to a quality model.
  • Cross-session memory and review code → file tickets from the Ask screen.
  • Mobile terminal — full repo shell from your phone over Tailscale.
  • Native desktop notifications + in-app update check; multi-project creds reuse and project delete.
v0.2.0 — Native macOS app Jun 2026

AgentHub leaves the terminal.

  • Ships as a signed & notarized .dmg — install it like any Mac app, no command line required.
  • Self-running app shell (Electron) bundling the web app + Jira worker; your data lives in Application Support.
  • Open on phone — a QR code and a stable Tailscale URL, with a Doctor connectivity check.
  • Login-shell PATH resolution so the Finder-launched app finds your CLIs (your agent, gh, tuist…).
v0.1.0 — Foundation Jun 2026

The mission-control foundation.

  • Live Jira board, Context Packs, agent worktrees, deep QA gate, Autopilot, Wiki + RAG, and Ask — all local-first on SQLite.
Hardening & Metrics Jun 7, 2026

Security, durability and visibility.

  • Optional shared-token auth on every page and API; Jira tokens encrypted at-rest (AES-256-GCM).
  • SQLite WAL + busy-timeout, Prisma Migrate, and a one-command VACUUM-INTO backup.
  • Metrics dashboard — outcomes, success / verify-green rates, diff-coverage, PRs, run time and token cost.
  • Vitest suites for the gate / state-machine / crypto logic, wired into GitHub Actions CI.
Deep QA, Knowledge & Autopilot Jun 6, 2026

The quality engine and the brain.

  • Deep QA engine — diff-coverage, quality gates + a bounded auto-fix loop, cross-model AI review and a plan-checker.
  • Security & dead-code scans (gitleaks, semgrep, periphery) plus a Quality Scorecard on every PR.
  • Project Wiki with local semantic search (Ollama bge-m3), Confluence import and attachments → context (videos → frames).
  • Ask — project Q&A grounded in your wiki, tickets and runs.
  • Autopilot — batch runs with a conflict-aware scheduler → a Draft PR per ticket, plus a realtime agent log.
Agent execution & worktrees Jun 4, 2026

Agents that edit real branches.

  • Full run lifecycle + a plan-review gate (approve, or request changes → it replans).
  • Per-ticket git worktrees (branch = the ticket key off develop); up to 3 runs in parallel.
  • Local Changes card — review the diff, commit (conventional) and open a PR when you're happy.
  • Works with whatever agent CLI you run — not locked to any one vendor.
Context Pack Jun 4, 2026

Give the agent the full picture.

  • Per-ticket Context Pack aggregating Jira, Confluence, Figma links and related repo history.
  • Token-budgeted, with per-source status and one-click copy.
Orchestration shell Jun 4, 2026

The mission-control foundation.

  • Live Jira board (six columns) synced by a standalone polling worker.
  • Ticket workspace, settings, an "only my tickets" filter, and SQLite persistence — zero infra.
