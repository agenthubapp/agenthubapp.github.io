v0.10.4 — the Cursor out-of-usage fallback actually fires now Jul 12, 2026

v0.10.3 — Cursor works even when your plan is out of usage Jul 12, 2026

Automatic fallback to Auto.

  • Automatic fallback to Auto — when your Cursor plan hits its usage limit (only the Auto model keeps working), AgentHub now detects it and retries once on Auto instead of failing every phase. Once the fallback fires, the rest of the run stays on Auto.
  • Honest errors — the limit error used to masquerade as "Cursor could not produce a structured plan" after burning four pointless retries, and Ask sessions died as a silent "exited with code 1". Now the pause says exactly what happened ("Your Cursor plan is out of usage — only the 'auto' model works. Pick 'auto' as the run model, or switch executor.") and stops retrying immediately.
  • Verified end-to-end against a replay of the real error stream from the field report.
v0.10.2 — Platform polish: sharper Inbox, smarter Dashboard, resilient Terminal Jul 12, 2026

Stopping a batch now really stops it.

  • Stopping a batch now really stops it — items paused on a question or a failure are cancelled too, instead of haunting the Needs Input column forever.
  • The Inbox stays honest — a superseded run's question disappears once a newer run takes over, batches that need attention (or hit their token budget) now appear as inbox items, and pending visual-snapshot reviews do too. Inbox polling is also dramatically lighter.
  • Nova-created batches are labeled — a Nova badge shows which batches she queued.
  • Date-range toggle — (30d / 90d / all) and a corrected PR count (manual PRs counted, rework runs no longer double-counted).
  • **New cards**: an Agents scorecard (the same track record Nova uses to pick executors), a retrieval-quality sparkline from the built-in eval history, and agent-discipline stats (plan deviations, lessons learned).
  • Today's briefing now mentions what Nova dispatched and finished.
  • The terminal survives network blips — it reconnects to the same shell with your scrollback intact (great on the phone over Tailscale).
  • Maestro regression suite — skips gracefully when no simulator/device is booted instead of reporting false regressions, and failing flows now explain themselves in the gate message.
  • Design relay port — can be changed from Settings and applies within a minute — no worker restart.
  • agenthubapp.github.io got a full redesign: a live animated run in the hero, an interactive product tour, and a Nova spotlight.
v0.10.1 — Ask levels up, Inspect & Design close their loops Jul 11, 2026

True multi-turn agent sessions.

  • True multi-turn agent sessions — follow-up messages in Agent mode resume the same CLI session (Claude Code, Codex, Cursor), so the agent remembers what it read and changed; falls back to a fresh session automatically if a session expired.
  • Tool-step timeline — a collapsible "🛠 12 steps · 3 edits · 1m 40s" timeline replaces the single activity line, streamed live and preserved in the transcript.
  • Per-file & per-hunk revert — every file block and every hunk in an agent diff gets its own revert control (validated with git apply --check so a stale diff can't corrupt the tree), plus "Restore to before this reply" via automatic pre-turn checkpoints.
  • Composer — slash commands (/review, /fix, /test, /clear) and a Stop button that keeps the partial answer.
  • Android live capture — the cockpit now screenshots a connected device/emulator on Android projects (adb), with platform-correct guidance; per-project Figma tokens are respected everywhere.
  • Re-inspect after rework — pins you flagged last time carry into the next review session (marked "from last review", one-tap Resolved); unresolved ones are prioritized in the rework feedback.
  • "Inspected ✓" board chip — see at a glance which review-ready tickets already passed manual inspection (a newer run invalidates it).
  • Faster queue loading (no longer loads every ticket's screenshots just to filter).
  • No more silent hangs — the page shows live "Plugin connected / not detected" status and refuses to start a run into an empty Figma channel with an actionable message; a Stop button cancels a running draw.
  • Iterate on a design — "Continue" modifies the frames you already drew instead of starting over; history entries show a rendered preview and an "Open in Figma" link.
  • Attach to ticket — a drawn design becomes the build-to-spec reference for that ticket's next agent run (and survives tracker syncs).
  • Relay hardening: port-conflict retry, health heartbeat, a new doctor check, and stale "running" sessions are swept.
  • Board no longer flashes the list view for board-preference users; switching projects can't leave a blank detail pane.
  • Nova now follows up on batch dispatches too, and picks the dispatch agent by recent track record while holding back tickets that would collide with in-flight work.
  • Android runs get scoped tests on the fast lane and a flake-retry (a test that passes on re-run no longer fails the gate).
  • Vietnamese UI now covers server-generated texts (needs-input asks, triage reasons, inbox details).
v0.10.0 — Nova becomes a teammate + a measurable context brain Jul 11, 2026

One-tap dispatch.

  • One-tap dispatch — Nova spots high-priority, agent-ready tickets and offers to assign them; one tap (in-app or on the notification's action button) starts fully-gated auto runs. Never merges anything by itself.
  • Speaks up when the app is closed — worker-side push engine with quiet hours (default 22–8), a strict never-repeat rule per suggestion, and escalating back-off when you tap "Not now" (24h → 3d → 7d).
  • Morning standup & weekly report — a daily "what needs you + what's ready" push (rephrased by your local LLM in Nova's voice, with a hard fact-preserving guard) and a Monday ROI summary (runs done, PRs merged, clean-merge rate, tokens).
  • Sentinel — warns about PRs open ≥48h, runs silent ≥30m, newly-muted baseline tests (with a one-tap "Fix it" that dispatches a repair run), and failing agents (open circuit breakers).
  • Evidence-based orchestration — the dispatch executor is chosen by recent track record (gate passes, clean merges, UI vs backend kind), risky tickets whose predicted files collide with in-flight worktrees/PRs are held back, and 2+ tickets dispatch as a conflict-aware Autopilot batch.
  • Audit thread — everything Nova does lands in a persistent "Nova" conversation with live run cards, so the whole story is reviewable in one place.
  • Retrieval eval harness — "Evaluate retrieval" on the Wiki page scores code search against real ground truth (the files each finished run actually changed): recall@5/@10, zero-recall tickets, config snapshot, and a recall trend recorded after every reindex.
  • Weight tuner — "Optimize weights" sweeps the dense/lexical blend over that same ground truth and applies the best weight instantly (no reindex).
  • Symbol-boost — a ticket naming an exact declared type now always surfaces the declaring file.
  • Auto-wiki — the local LLM writes concise per-module wiki pages from your code, refreshed incrementally as modules change; never touches human-written pages.
  • Structural context — run context packs now include 1-hop import/type-reference neighbors of the top code hits (the callers/callees semantic search misses).
  • Android verify parity — per-test failures are now mined from gradle's JUnit XML, so the pre-existing-failure baseline finally works on Android (one known-bad test no longer blocks every auto run), and auto-fix feedback stops telling Android agents to write Swift tests.
  • Unconfigured build stack can no longer ship zero-QA PRs — it blocks the gate and asks you to configure Settings → Build stack.
  • Doctor gains an Android section — (SDK, adb, device/AVD, JDK).
  • Per-project repo path everywhere — codemap, worker terminal, merge-watch, Ask/chat pins and more no longer resolve the wrong project's repo in multi-project installs; a project without a repo now says so instead of silently using another project's.
  • Smarter agent-unavailable pauses — a paused run now offers real "Switch to X / Wait and retry" actions instead of generic options; more agent question formats are recognized.
  • Missing figmaApiToken migration — fresh installs and migrate-deploy upgrades no longer crash on project queries.

v0.9.0 — Understand before you ship Jul 6, 2026
