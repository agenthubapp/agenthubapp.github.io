Changelog

What's new in AgentHub

A running log of releases, features and fixes. Newest first.

v0.7.3 — Edit projects & fixed setup dialog Latest Jul 3, 2026

Edit project.

  • Edit project — a pencil button in the project switcher (and the "Jira not configured — set up" link in the header) opens the project for editing: name, Jira credentials, repo, base branch and verify config. No more re-creating a project just to add Jira.
  • Saved token stays safe — leaving the API-token field blank keeps the stored token; "Test connection" works against the saved token too.
  • Setup dialog fixed — the Add/Edit project dialog no longer renders clipped under the glass header; it is properly centered on every page.
  • Delete any project — the last remaining project can now be removed too; AgentHub returns to first-run setup.
v0.7.2 — Honest agent errors, custom gateways & debug bundle Jul 3, 2026

No more garbage answers in chat.

  • No more garbage answers in chat — when an agent CLI fails (out of credit, expired login, stray terminal codes), Ask now fails over to the next agent and reports the real cause with a fix hint, instead of posting the raw error as a reply.
  • Actionable auth & billing errors — "Login canceled", "Credit balance is too low" and similar failures now pause runs with the exact next step (re-login command or where to fix the key), in both the run panel and chat.
  • Codex behind a company AI gateway now works — if your Codex CLI goes through a LiteLLM / corporate gateway, paste that gateway's token in Settings → AI Connection and AgentHub uses it correctly (it no longer rejects gateway tokens as invalid OpenAI keys).
  • Debug bundle — one click in Settings → System check downloads a diagnostics file (agent status, recent failed-run logs — secrets removed) that you can attach to a bug report instead of screenshots.
  • Desktop app: advanced settings file — power-user options that don't have a UI yet can be set in a plain text file the app creates for you at ~/Library/Application Support/AgentHub/env; error messages and Settings now point to this exact file.
v0.7.1 — Ask 2.0 + platform hardening Jul 3, 2026

Ask 2.0.

  • Ask 2.0 — real-time streaming answers, rich markdown with highlighted code + clickable file references, @file/@folder context mentions with visible context pills, and reviewable diffs with one-click revert in Agent mode
  • Run-in-chat — type "fix ticket KEY" in Ask and the whole run (phases, questions, diff, summary) happens live inside the conversation
  • Inspect redesigned — one-step review cockpit with background sim capture, pins in overlay/compare modes, and mobile support
  • Bring your own AI endpoint — the local "brain" (retrieval rerank, briefings) now speaks OpenAI-compatible; default stays local Ollama, or point it at LM Studio/OpenRouter/any gateway in Settings → AI models
  • Light mode — light/dark/auto theme, flash-free, desktop chrome included
  • Sharper agents — Spec-Contract scope enforcement on every diff, clearer "needs your input" prompts with one-tap options, smarter wiki (learning dedup + health panel)
  • Fixes — pop-out panel no longer breaks at small heights; slimmer desktop app (no stray mobile shell)
v0.7.0 — Inspect: Figma vs sim review Jun 27, 2026

Inspect cockpit.

  • Inspect cockpit — Side-by-side Figma design vs live simulator with split, overlay, and slider compare; zoom, checklist, sign-off, or send pins to the agent for rework
  • Review queue — Ready Review tickets surface on Inspect home; resume draft sessions without slow reloads
  • Paste PNG from Figma — Copy a frame in Figma (⌘C) and paste into Inspect (⌘V) — instant reference without waiting on Figma API
  • Figma in Settings — Encrypted personal access token stored in the app database; smart frame picker skips toolbar strips and prefers phone-sized screens matched to ticket hints (e.g. SOS)
  • Desktop-first Inspect — Hidden from mobile tab bar; full cockpit on Mac where side-by-side review actually works
  • Settings refresh — Reorganized System, Agents, and Project settings with Figma connection test
v0.6.0 — Faster runs, reliable autopilot Jun 26, 2026

Manual fast lane.

  • Manual fast lane — diff-first iteration: see changes right after execute without waiting for full verify (on by default)
  • Autopilot reliability — batch items no longer stuck as "running"; auto runs proceed past untagged plan questions; Jira dep deadlocks fixed
  • Smarter verify defaults — autopilot fast lane off by default so auto PRs go through build/test; batch advisory PR opens draft PR on gate failures instead of pausing the whole batch
  • UI capture scope — Figma matched to the right screen; wrong-screen captures block auto runs
  • Pipeline knobs — manual fast, unattended batch, batch advisory PR in Settings
v0.5.5 — AI connection without .env Jun 25, 2026

AI connection.

  • AI connection — Paste OpenAI/OpenRouter keys in Settings; stored encrypted in the app database (no .env for DMG users)
  • Test connection — One-tap check that Codex/Cursor and the default model are ready before you run
  • Pre-run gate — Blocks bad runs early with plain Vietnamese/English errors instead of failing mid-planning
  • Run instructions — Extra instructions (Confluence, Figma, day/night rules) now flow into the context pack
  • Skip verify fix — Express + skip-verify prefs survive plan approval on manual runs
  • Small-bug preset — One click: Express + skip verify + Cursor for tiny fixes
v0.5.4 — Agent reliability & Needs Input UX Jun 25, 2026

Ask Agent.

  • Ask Agent — No more fake "Updated N files" replies; greetings stay read-only; only real model text or a clear CLI error
  • Plan/exec failures — Codex/Devin stderr (e.g. Windsurf permission denied) surfaces instead of silent "(no plan output)"
  • Needs Input — Hero panel with actionable guidance when a run pauses for you
  • Settings / Doctor — Warns when Codex default model may fail headless; adds default-model plan probe
  • Run stream — Smarter polling hooks and SSE fallback for live run panels
  • Mission control — Workspace, inspector, and activity panel polish
v0.5.3 — Pop-out polish Jun 25, 2026

Pop-out scroll.

  • Pop-out scroll — Runs, Preview, Inbox, and Terminal tabs scroll correctly in the pinned panel (Electron + browser PiP).
  • Auto-dock countdown — Slim violet progress line under the header when the panel auto-tucks after 5s; easier to see without feeling heavy.
v0.5.2 — Claude Code + Devin fix Jun 25, 2026

Claude Code.

  • Installed agents only — Settings and run pickers list only CLIs found on PATH (Codex, Devin, Claude Code, Cursor)
  • Board filter — new "High priority" chip for urgent Jira tickets
v0.5.1 — Electron PiP edge dock Jun 25, 2026

Pop-out docks like macOS PiP — edge tab, drag, snap.

  • Pop out → dock — opens as a slim edge tab immediately (no full-panel flash).
  • Drag along edges — slide on left/right/top/bottom; release anywhere and it snaps back to the nearest edge (system PiP behaviour).
  • Smart peek — auto-expands on plan ready / needs input, then tucks back after 5s unless you interact.
  • Tap chevron to expand; drag the header when expanded.
  • pack:install — one command to build and copy the .app into /Applications.
v0.5.0 — Cursor planning & agent clarity Jun 25, 2026

  • Mobile ticket UX — compact header, sticky tabs, more scroll room on Run/Changes/Context.

v0.4.0 — Mission Control & agent intelligence Jun 22, 2026

A control room built for how you actually ship.

  • Mission Control workspace — board or list, split detail + live preview + inspector, run cockpit, and per-ticket base branch.
  • macOS-native shell — vibrancy, SF Pro, Liquid Glass sidebar, grouped nav, ⌘K command palette, and shortcut cheatsheet.
  • Today briefing — a daily feed of what needs you, wins, and an LLM-written digest from your project activity.
  • Agent Activity cockpit — live routing, activity feed, sub-agent tree, and phase timeline so runs are observable, not opaque.
  • Cursor CLI as a third executor alongside Codex and Devin, with smart fast/deep routing per phase.
  • Ask upgrades — cross-session memory, review code → file Jira tickets (AgentHub POSTs issues directly), visual answers with screenshot + Figma.
  • Conflict resolution — PR mergeable state on GitHub, push/force-push, manual 3-way hunk picker, and agent-assisted resolve with marker guards.
  • Spec Steward — decision book, spec contract, and auto-resolve for open plan questions so agents stay on-spec across tickets.
  • Learn-from-edits loop — when you edit a merged PR, AgentHub distills corrections into wiki learnings for future runs.
  • Maestro regression sentinel, PiP pop-out panel, Android stack support, and ~25–50× faster warm RAG retrieval via vector cache.
