Why AgentHub

Not just another agent runner.

Plenty of tools can launch an agent. AgentHub is the part that makes what it produces actually mergeable.

Mission Control, not just a board

Board or list view, a split detail workspace with live preview, run cockpit, and an inspector — plus ⌘K to jump anywhere. One screen to monitor every ticket.

Inspect before you ship

Ready for review? Open Inspect: paste a frame from Figma, compare it beside the live sim (split, overlay, or slider), drop pins on the build, then sign off or send pins straight to the agent.

A QA gate that self-heals

Build, test, lint, diff-coverage, a cross-model AI review and security scans run on every change. When something fails, AgentHub feeds the errors back and the agent retries — before it ever pings you.

Research-grade RAG, 100% local

Hybrid retrieval, LLM reranking, query expansion and contextual chunks — powered by local Ollama models. Your code and context never leave your Mac.

A backlog in, a stack of PRs out

Autopilot's conflict-aware scheduler runs tickets in parallel where they're safe, sequential where they collide, and opens a Draft PR for each.

It builds to your design

For UI work it extracts a grounded spec from Figma at plan time, builds to it, then scores the running screen against your agreed tokens — not just a screenshot.

The loop

One ticket in. One reviewed PR out.

AgentHub runs the same disciplined pipeline a senior engineer would — for every ticket, automatically.

Sync

Your Jira board streams in live — assigned tickets, every ~45 seconds.

Context

A Context Pack pulls Jira, Confluence, Figma, repo history and attachments.

Execute

The agent plans, you approve, then it edits an isolated per-ticket git worktree.

Verify

A deep QA gate runs build · test · lint · coverage · AI review · scans — with auto-fix.

Ship

Green verify opens a Draft PR with a quality scorecard. You review and merge.

Capabilities

Everything you need to ship like a team of one.

A complete control room for AI-assisted delivery — from the board to the merged PR.

Inspect — Figma vs build

A focused review cockpit for Ready Review tickets: Figma beside the live simulator, overlay or slider compare, zoom, paste a frame from Figma (⌘C/⌘V), pin issues on the build, sign off, or send pins to the agent for rework.

Mission Control workspace

Board or list, then drill into a ticket: live run cockpit, diff preview, context, local changes, and an optional inspector — without leaving the flow.

Live Jira board

Six run- and PR-aware columns, refreshed continuously. Pause agents, cancel a run, or approve a plan straight from the card.

Context Packs

Aggregates Jira, Confluence, Figma, related repo history and attachments — videos become frames — into a token-budgeted, copyable brief.

Bring your own agent

AgentHub drives Codex, Devin, or Cursor CLI — each in its own per-ticket git worktree. Smart routing picks fast vs deep models; up to three tickets in parallel.

Plan-review gate

The agent writes a structured, grounded plan first. Approve it — or request changes and it replans until the plan holds up.

Deep QA gate

Build, test, lint, a diff-coverage threshold and a cross-model AI review — with a bounded auto-fix loop that feeds failures back to the agent.

Security & dead-code scans

gitleaks for secrets, semgrep for conventions, periphery for dead code. Leaked secrets and HIGH findings block automated runs.

Autopilot batches

Queue a stack of tickets; a conflict-aware scheduler runs them parallel-where-safe, sequential-where-they-touch, and opens a Draft PR for each.

Project Wiki + RAG

Local embeddings (Ollama bge-m3), hybrid retrieval with LLM rerank, Confluence import — plus a learnings loop that remembers past tickets.

UI-vs-Figma proof

Installs the build on a simulator, drives it with mobile-mcp, and scores the result against your agreed Figma design tokens.

Metrics dashboard

Run outcomes, success and verify-green rates, diff-coverage, PRs opened and merged, average run time and token cost — all in one view.

Ask your project

Q&A grounded in wiki, tickets, and code — with fast/deep routing, cross-session memory, visual answers (screenshot + Figma), and review code → file Jira tickets.

Today briefing

A daily home feed: what needs you (approve plans, answer pauses, stale PRs), recent wins, and an LLM-written briefing from your project's activity.

Agent Activity cockpit

Live routing decisions, an activity feed, sub-agent tree, and a phase timeline — so you always know what the agent is doing and why.

Any stack

iOS/Xcode works out of the box. Bring your own build, test and lint commands for web, backend or Android — same gate, same pipeline.

Mobile + PiP

Phone access over Tailscale — approve plans from push, watch runs, even a repo terminal. Pop out a floating PiP panel on desktop for always-on monitoring.

Conflict resolution

PR-level conflict detection, push/force-push when rebased, a manual 3-way hunk picker, or resolve with agent — with a guard that refuses to commit leftover markers.

Quality engine

A QA gate that doesn't wave code through.

Every run is held to the same bar — the one you'd hold a teammate to. When it fails, AgentHub feeds the exact errors back to the agent and tries again, before it ever asks for your time.

  • Build · test · lint on the real toolchain, with per-test results mined from the result bundle.
  • Diff-coverage on the lines you actually changed — agents are prompted to write the tests.
  • Cross-model AI review that must clear with no HIGH findings.
  • A quality scorecard posted on the PR, plus a commit status and work summary on the ticket.

Autopilot

Hand it a backlog. Get back a stack of PRs.

Select a batch of tickets and AgentHub builds a file-overlap conflict graph, then schedules the work parallel-where-safe and sequential-where-they-collide. It auto-approves clean plans and only stops for you when a ticket genuinely needs input.

  • Readiness triage before anything runs — vague tickets surface early.
  • Conflict-aware scheduling so parallel tickets never corrupt each other's build.
  • A needs-input inbox for open questions and gate failures — pause or stop the batch anytime.

Knowledge

  • A learnings loop — each passing run becomes institutional memory for the next.

Local-first by design

Your tickets, code and keys never leave your Mac.

AgentHub is a single-user power tool on purpose — no multi-tenancy, no accounts, no servers to run. It's built to make one developer ship like a whole team, privately.

Runs entirely on your Mac

SQLite plus a polling worker. No cloud database, nothing to provision or pay for.

Your code stays local

Agents edit local git worktrees. Only you push branches and open the PRs.

Secrets encrypted at-rest

Jira tokens are sealed with AES-256-GCM; the key is generated locally and locked down.

Optional token auth

Require a shared token on every page and API when you expose it beyond localhost.

Built on a modern, boring-in-the-best-way stack.

Get started

Up and running in three commands.

Clone, set up, and launch the board with its Jira poller — then let the Welcome guide do the rest.

zsh — AgentHub
# clone & install $ git clone … && cd AgentHub $ npm install # one-command setup (.env.local + dev.db) $ npm run setup # web (:3000) + Jira poller together $ npm run dev:all
1

Connect Jira

Add an API token and AgentHub syncs the tickets assigned to you — no token, it runs on seed data.

2

Point at your repo

Pick the folder, then choose ios-xcode or a custom stack with your own build/test/lint.

3

Assign an agent

Sign in to your agent CLI (e.g. codex or devin) plus gh from the System check, then run a ticket end to end.

First run? Follow the Welcome guide

It walks you through a System check that detects and installs every dependency for you.
